# Product Requirement Document: Local Card Management App

## 1. Core Views
- **Dashboard View:** Must display user cards in a responsive grid as visual credit card components.
- **Card Status Toggles:** Users must be able to switch active cards to "blocked" via a UI element.

## 2. Strict UI Rules
- Active cards must feature a green glowing boundary dot.
- Blocked cards must have a grayscale overlay with an opacity of 50%.

## 3. Data Constraints
- Card spending limits cannot exceed a maximum ceiling of $50,000.
